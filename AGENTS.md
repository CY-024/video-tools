## 项目大概是做什么的

这是一个面向自媒体视频创作的网页工具原型，主题是“爆款解说视频工作台”。

它看起来像一个 AI 视频创作后台，主要围绕这些事情：

- 学习热门视频文案结构。
- 根据需求生成解说文案。
- 把文案和画面片段做匹配。
- 模拟剪映一样的剪辑工作台，包括媒体库、播放器、字幕、音频和时间线。

目前它更像“可交互的前端展示原型”，不是完整的真实剪辑软件。也就是说，页面能点、能切换、能展示导入效果，但很多 AI 生成、视频处理、联网搜索等功能还是演示状态。

## 技术栈

- 使用 React 做页面。
- 使用 Vite 启动和打包项目。
- 使用 lucide-react 提供图标。
- 项目是纯前端，没有看到后端接口、数据库或服务器业务代码。

给 Rowan 解释时可以这样说：

- React：用来搭网页界面的工具。
- Vite：让网页在本地快速跑起来、最后打包成可发布文件的工具。
- lucide-react：一套现成的小图标库。

## 主要文件结构

- `package.json`：项目配置，里面写了怎么启动、怎么打包、用了哪些工具。
- `package-lock.json`：依赖锁定文件，记录安装包的具体版本，通常不要手动改。
- `index.html`：网页入口，负责挂载 React 页面。
- `src/main.jsx`：React 启动入口，把 `App.jsx` 渲染到网页里。
- `src/App.jsx`：主要页面逻辑和界面内容都在这里。
- `src/App.css`：页面样式，控制颜色、布局、按钮、面板、剪辑时间线等视觉效果。
- `dist/`：打包后的成品目录，由 `npm run build` 生成，一般不要手动改。
- `node_modules/`：安装的第三方依赖目录，一般不要读也不要手动改。
- `.gitignore`：告诉 Git 哪些文件不用记录。

## 页面功能模块

`src/App.jsx` 里有 4 个主要模块：

1. `learn`：爆款文案学习
   - 展示热门视频样本。
   - 展示文案提取结果。
   - 点击学习按钮后会用前端状态模拟“学习中”和“已学习”。

2. `write`：文案生成
   - 展示创作需求。
   - 模拟导入原视频或联网搜索片源。
   - 展示生成后的解说文案和 SRT 字幕草稿。

3. `match`：智能匹配剪辑
   - 展示原视频导入区域。
   - 展示 B-roll 推荐。
   - 展示文案和画面一对一匹配表。
   - 底部会出现多轨时间线。

4. `edit`：剪辑工作台
   - 模拟类似剪映的工作区。
   - 左侧是媒体库，可以导入视频、音频、SRT 字幕文件。
   - 中间是播放器预览。
   - 右侧是画面和字幕调节区。
   - 下方是时间线。
   - “示例导入”按钮会添加演示素材，不是真的处理视频。

## 当前已知问题

- `index.html` 和 `src/App.jsx` 里的大量中文显示为乱码。
- `AGENTS.md` 原来也出现了乱码，本文件已经重新用中文整理。
- 乱码大概率是文件编码问题，类似“中文用错了打开方式”，不是页面逻辑本身的问题。
- 虽然文字乱码，`npm run build` 目前仍然可以成功打包。
- `EditView` 接收了字幕大小、配音等参数，但当前函数内部没有实际使用这些参数。
- 项目里没有自动化测试脚本。
- 页面设置了较大的最小宽度，偏桌面端，不太适合手机小屏幕。

## 修改代码时的注意事项

- 优先修改 `src/App.jsx` 和 `src/App.css`。
- 不要手动修改 `node_modules/`。
- 不要手动修改 `dist/`，除非明确是在处理打包结果。
- 修改文字时要注意文件编码，建议保持 UTF-8，避免中文再次变乱码。
- 如果改了界面，最好运行 `npm run build` 确认没有打包错误。
- 如果要给 Rowan 解释样式变化，要说“页面看起来会怎么变”，比如：
  - “左边菜单会变窄一点。”
  - “按钮颜色会更明显。”
  - “时间线区域会更像剪映。”
